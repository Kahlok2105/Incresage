🔍 Breakthrough Debug Analysis

✅ Issues Found and Fixed

  • Issue 1: Qi not being reduced on failure - FIXED
    • The problem was in the canAttempt condition in useGameLoop.ts
    • It required full Qi (state.qi >= nextRealm.qiRequired) but the UI allowed attempts with 50% Qi
    • This caused the function to return early, skipping the state update (including Qi reduction)
    • Fix: Changed condition to state.qi >= nextRealm.qiRequired / 2 to match UI

🔍 Issue 2 Analysis: Breakthrough Chance

  • Expected Behavior:
    • Base success rates: Mortal→Qi Condensation (90%), Qi Condensation→Foundation (80%)
    • Actual chance = baseSuccessRate × (currentQi / requiredQi)
    • If you have 50% Qi, your chance is 50% of the base rate
  • Possible Explanations:
    • UI shows maximum possible chance: The UI might show the chance when you have full Qi (e.g., 80%), but your actual chance with 50% Qi would be 40%
    • Bad luck streak: Even with 50% chance, there's a (0.5)^20 = 0.00095% chance of failing 20 times in a row
    • Qi level lower than expected: If you have less than 50% Qi, your chance would be even lower

🎯 How to Verify the Fixes

  1. Test Qi reduction on failure:
    • Accumulate some Qi (e.g., 500/1000)
    • Attempt a breakthrough and fail
    • Check if your Qi is reduced to 250 (50% of 500)
    • You should see a red failure notification
  2. Check your actual chance:
    • Look at your current Qi vs required Qi
    • Calculate: baseRate × (currentQi / requiredQi)
    • Example: 80% base × 50% Qi = 40% actual chance
  3. Test with full Qi:
    • Fill your Qi to maximum
    • Attempt breakthrough - you should have the full base rate chance
    • On success, you should see a green success notification
