🧪 Breakthrough Spirit Stone Test

📋 Test Description

This test verifies that breakthrough failures now consume spirit stones, adding risk/reward balance to the game.

Expected behavior: When a breakthrough fails, both Qi (50% reduction) AND spirit stones should be consumed.

🔧 Test Setup

Initial state for testing:

  • Current Realm: Mortal (Index 0)
  • Qi: 500/1000 (50% of required 1000 for next realm)
  • Spirit Stones: 10 (required: 10 for next realm)
  • Breakthrough Chance: ~45% (90% base × 50% Qi)

🎯 Test Scenarios

  1. Failure Test: Force a breakthrough failure and verify spirit stones are consumed
  2. Success Test: Force a breakthrough success and verify spirit stones are consumed
  3. Edge Case: Test with insufficient spirit stones

📊 Expected Results

Scenario Before After Result
Failure Qi: 500, Stones: 10 Qi: 250, Stones: 0 ✅ Stones consumed
Success Qi: 1000, Stones: 10 Qi: 0, Stones: 0, New Realm ✅ Stones consumed

✅ Implementation Details

The change was made in incresage/src/hooks/useGameLoop.ts:

// Before (failure case):
return {
  ...prev,
  qi: Math.max(0, prev.qi * 0.5),
};

// After (failure case):
return {
  ...prev,
  qi: Math.max(0, prev.qi * 0.5),
  spiritStones: prev.spiritStones - nextRealm.stonesRequired,
};

This ensures that both successful and failed breakthroughs consume spirit stones, creating a proper risk/reward mechanic.

🎮 Game Design Impact

  • Risk/Reward Balance: Players now have to consider whether to attempt breakthroughs with lower chances, as failure has a real cost
  • Resource Management: Spirit stones become more valuable and require better planning
  • Strategic Depth: Players may choose to farm more spirit stones before attempting risky breakthroughs
  • Consistency: Both success and failure now have resource costs, making the mechanic more intuitive

📝 Test Notes

To manually test this in the game:

  1. Accumulate some Qi and spirit stones
  2. Attempt a breakthrough with a low chance (e.g., 50% Qi)
  3. If it fails, check that your spirit stones were consumed
  4. Verify that your Qi was reduced by 50%
  5. Try again with full Qi to see the success case
